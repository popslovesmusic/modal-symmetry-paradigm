MSP is composed of four architectural elements:

### 4.1 Modal Axes

Orthogonal (or quasi-orthogonal) directions of possible variation.  
These are the proto-dimensions that meaning will eventually inhabit.

### 4.2 Modal Symmetry Classes

Each symmetry class defines a family of allowed transformations.  
This is analogous to:

* symmetry groups  
* Lie group generators  
* coordinate transformations  
* basis vectors

### 4.3 Modal Invariants

Every mode has rules that cannot be violated.  
These invariants form the “laws of structure” that MSC later enforces.

### 4.4 Proto-Operators (δ, Φ, Π)

These arise naturally within MSP:

* δ — Deviation: the ability to detect difference  
* Φ — Projection: the ability to map raw signals into modal space  
* Π — Evaluation: the ability to test projected forms for consistency

These are not semantic operators yet.  
They are modal operators — the minimum machinery for modal coherence.

# 8\. MSP Invariants

MSP requires several universal invariants for the system to be coherent:

1. Modal Closure  
   Modes must fully cover all possible variations.  
2. Modal Consistency  
   Modes must not contradict each other.  
3. Modal Transformability  
   Any mode must be convertible into any other via allowed transformations.  
4. Modal Stability  
   Modes must not collapse spontaneously.  
5. Modal Sufficiency  
   The modes must be adequate to generate the full MBC Tier system.

Without these invariants, the system cannot support meaning.
